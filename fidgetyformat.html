📄 .fidget File Format & Examples

This page covers the core fields of the .fidget format and shows two ready-to-load examples: a toggle and a spinner.

🔨 Schema Versions

  • 1.0: The current version. Documented below.

🔑 Common Fields

  • type : "toggle" | "spinner"
  • name : Display name for your fidget
  • imageA : Base64-encoded image for default state
  • sound (optional) : Base64-encoded audio for click/interaction
  • schemaVersion (optional but strongly recommended) : Schema Version (e.g. "0.9")
  • author (optional) : Creator of the fidget (e.g. "Con and Sneaky")

🎚️ Toggle-Type Fields

When type is "toggle", you must also supply:

  • imageB : Base64 image for the alternate state

🌀 Spinner-Type Fields

When type is "spinner", you may add:

  • velocity (optional) : Angular velocity for the physics engine to push the Fidget Spinner with (number)

📦 Examples

1. Toggle Example: “Clicky Button”

{
  "type": "toggle",
  "name": "Clicky Button",
  "imageA": "data:image/png;base64,iVBORw0KGgoAAAANS…",
  "imageB": "data:image/png;base64,SU5BTkkgRk9SIElNQUdFUjEi…",
  "sound": "data:audio/mp3;base64,//uQxAB…"
}

2. Spinner Example: “Retro SpinX”

{
  "type": "spinner",
  "name": "Retro SpinX",
  "imageA": "data:image/png;base64,AAABAAEAICAAAAE…",
  "speed": 40,
}
